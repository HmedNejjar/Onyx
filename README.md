# Onyx — GPT-Style Language Model

A from-scratch GPT implementation in PyTorch. Trains a causal transformer decoder on text data and generates continuations from a prompt using combined top-p/top-k sampling with repetition penalty.

---

## What It Does

- Tokenizes raw text using the GPT-2 BPE tokenizer (50,257 vocab)
- Trains a 12-layer transformer decoder (768-dim, 12 heads) on next-token prediction
- Generates text autoregressively using combined top-p/top-k sampling with a repetition penalty

---

```
└── requirements.txt
```

## What Each File Does

**`model.py`** — `Onyx(nn.Module)`  
Token + positional embeddings → 12 transformer blocks → LayerNorm → linear projection to vocab logits. `generate()` tokenizes a prompt and samples tokens one at a time using `_sample_top_p_top_k()`, which applies repetition penalty, top-k filtering, top-p (nucleus) filtering, and temperature scaling in sequence. Raises `ValueError` if the prompt encodes to zero tokens.

**`TransformerBlock.py`** — `TransformerBlock(nn.Module)`  
Pre-norm decoder block. Flow: `LayerNorm → MultiheadAttention → Dropout → Residual`, then `LayerNorm → FFN (Linear → GELU → Linear → Dropout) → Residual`. The causal mask is a boolean upper-triangular matrix passed in from the model (`torch.triu(..., diagonal=1).bool()`), compatible with PyTorch's `nn.MultiheadAttention`.

**`Dataloader.py`** — `GPTDataset(Dataset)`  
Memory-maps a `.npy` token file via `mmap_mode='r'`. `__getitem__(i)` returns `(tokens[i*stride : i*stride+seq_len], tokens[i*stride+1 : ...+1])` — input and target shifted by 1 for next-token prediction. Dataset length is `(total_tokens - seq_length) // stride`.

**`precompute.py`**  
Downloads WikiText-103 via HuggingFace `datasets`, tokenizes with GPT-2 tokenizer (skipping blank lines), saves `wikitext_train.npy` and `wikitext_val.npy` as `int32` arrays. Run once before training.

**`train.py`**  
Training loop with mixed precision (`GradScaler` + `autocast`), gradient accumulation (effective batch = `BATCH_SIZE × ACCUMULATION_STEPS`), gradient clipping (`max_norm=1.0`), and validation after each epoch. Saves checkpoint to `best_model.pth` when validation accuracy improves. Metrics logged: train loss, train accuracy, val loss, val accuracy.

**`main.py`**  
Interactive inference script. Loads `best_model.pth`, prompts the user for input, and prints a 50-token continuation. Runs under `torch.no_grad()` in eval mode.

## How to Run

**1. Tokenize data** (once):
```bash
python GPT/precompute.py
```
Outputs `Data/wikitext_train.npy` and `Data/wikitext_val.npy`.

**2. Train:**
```bash
python GPT/train.py
```
Saves `best_model.pth` when validation accuracy improves.

**3. Generate (interactive):**
```bash
python GPT/main.py
```

Or programmatically:
```python
from GPT.model import Onyx
import torch

model = Onyx()
model.load_state_dict(torch.load("best_model.pth"))
model.eval()

with torch.no_grad():
    print(model.generate("Once upon a time", max_tokens=100, top_p=0.95, top_k=50, temp=0.7, repetition_penalty=1.2))
```

## Key Hyperparameters

| Parameter | Default | Notes |
|---|---|---|
| `VOCAB_SIZE` | 50,257 | GPT-2 BPE vocab |
| `CONTEXT_LEN` | 1024 | Max sequence length |
| `EMB_SIZE` | 768 | Embedding dimension |
| `NUM_HEADS` | 12 | Attention heads |
| `NUM_LAYERS` | 12 | Transformer blocks |
| `BATCH_SIZE` | 2 | Per-step GPU batch |
| `ACCUMULATION_STEPS` | 8 | Effective batch = 16 |
| `LR` | 1e-4 | AdamW learning rate |
| `STRIDE` | 256 | Sliding window step (`CONTEXT_LEN // 4`) |
| `DROPOUT` | 0.1 | Applied to attention, FFN, and residuals |

### Generation Parameters (`generate()` defaults)

| Parameter | Default | Notes |
|---|---|---|
| `max_tokens` | 100 | Tokens to generate |
| `top_p` | 0.95 | Nucleus sampling threshold |
| `top_k` | 50 | Top-k candidates kept before nucleus filter |
| `temp` | 0.7 | Temperature; lower = more deterministic |
| `repetition_penalty` | 1.2 | >1.0 penalises previously seen tokens |

## Architecture Notes

- **Pre-norm**: LayerNorm is applied before each sub-layer (attention, FFN) rather than after, improving training stability.
- **Causal mask**: Boolean upper-triangular matrix (`triu(..., diagonal=1).bool()`) prevents tokens from attending to future positions.
- **Sampling**: `_sample_top_p_top_k()` applies repetition penalty → top-k filter → top-p (nucleus) filter → temperature scaling → multinomial sample, in that order.
- **Weight tying**: Token embedding and output projection are not tied (separate `nn.Embedding` and `nn.Linear`).
